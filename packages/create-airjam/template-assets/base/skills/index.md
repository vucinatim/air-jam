# Skills Index

These local skills teach how to work in this repo.

Do not read every skill for every task.

Pick the one that matches the current problem.

## Core Skills

1. `plan-ledger`
   Use when starting or progressing non-trivial work and you need to keep `plan.md` and `suggestions.md` disciplined.
2. `airjam-docs-workflow`
   Use when you need canonical guidance and need to know whether local docs or hosted docs should be consulted.
3. `game-architecture`
   Use when creating systems, moving code, or deciding where logic should live.
4. `game-state-and-rendering`
   Use when changing stores, frame logic, refs, React state, or rendering integration.
5. `controller-ui`
   Use when building or reviewing controller surfaces, including the shared controller shell contract, lifecycle actions, and icon choices for controller and gameplay UI.
6. `debug-and-test`
   Use when debugging with the unified dev log stream, adding diagnostics, or writing tests.
7. `host-surface`
   Use when building or reviewing the host shell, host-lobby join contract, active game viewport, overlays, and room-facing presentation.
8. `prefab-authoring`
   Use when creating reusable gameplay objects or scene presets that should stay declarative, configurable, and tool-friendly.
9. `r3f-3d-scene`
   Use when building or reviewing 3D Air Jam gameplay with React Three Fiber, lighting, shadows, grounding, particles, and physics.
10. `canvas-2d-gameplay`
   Use when building or reviewing 2D canvas gameplay, render/simulation separation, scaling, SVG assets, and lightweight collision patterns.

## Rule

If the task does not match a skill boundary, do not force it.

Use the smallest relevant guidance.
